一个面向本地门店的静态前端演示页，把获客诊断、内容增长、车主转化和经营看板串成一条可以讲清楚的业务方案。

线上演示 ↗GitHub ↗

背景

门店经营方案需要被快速看懂，而不是只写在文档里。

这个 demo 把试点目标、获客诊断、内容生产、转化路径和后续运营拆成可浏览的页面结构，让方案更容易被客户或团队理解。

我负责

用静态前端把商业叙事整理成可上线页面。

  • 用 HTML、CSS 和 JavaScript 完成页面搭建和视觉层级。
  • 把门店经营方案拆成定位、AI 搜索、能力、获客、转化、后续等模块。
  • 通过 GitHub Pages 形成可直接访问的线上演示路径。

结果

它更适合展示页面组织、商业表达和静态部署能力。

目前这页已经换成真实线上 demo 截图，不再使用仓库 Open Graph 小图。
